# Design System Library — Index

**129 portable design systems** (`DESIGN.md` format).

Imported from [`nexu-io/open-design`](https://github.com/nexu-io/open-design) (Apache-2.0).
See [`ATTRIBUTION.md`](./ATTRIBUTION.md) for licensing.

## Cómo usar

Estos sistemas son **referencia para discutir, no plantilla a clonar**.
Cuando el usuario diga "quiero estilo X":

1. Buscar `library/<slug>/DESIGN.md` en este índice
2. Leerlo completo antes de generar tokens
3. Preguntar al usuario qué adaptar (no clonarlo tal cual)
4. Usarlo como hint en el system prompt, no como output literal

Si todos los Forge-built SaaS terminan viéndose como Linear o Stripe, fallamos.
El objetivo es **inspirar**, no **replicar**.
