# /kanban — Tablero Kanban por User Story

Genera y actualiza un tablero Kanban visual basado en las User Stories del proyecto. Permite trackear el progreso de desarrollo de cada story a través de las fases del pipeline.

## Instrucciones

### Paso 1: Detectar User Stories

Busca User Stories en el proyecto:

1. **Primero:** Buscar `USER-STORIES-*.md` en la raíz
2. **Segundo:** Buscar `BLUEPRINT-*.md` y extraer la sección de User Stories
3. **Tercero:** Buscar `PIEZA-*.md` y extraer fases/tasks
4. **Si no hay ninguno:** Preguntar al usuario qué quiere trackear

### Paso 2: Definir Columnas

Las columnas del Kanban siguen el flujo de Forge:

| Columna | Significado | Emoji |
|---------|-------------|-------|
| **Backlog** | Definida pero no iniciada | 📋 |
| **Planificación** | En fase de diseño/spec | 📐 |
| **Desarrollo** | Código en progreso | 🔨 |
| **QA** | Testing y validación | 🧪 |
| **Review** | Code review o design review | 👀 |
| **Done** | Completada y deployada | ✅ |

### Paso 3: Generar Tablero

Crear el archivo `KANBAN.md` en la raíz del proyecto:

```markdown
# KANBAN — [Nombre del Proyecto]

> Última actualización: [fecha y hora]
> Progreso: [X/Y] stories completadas ([Z]%)

---

## 📊 Vista General

```
📋 Backlog    │ 📐 Planning  │ 🔨 Dev       │ 🧪 QA        │ 👀 Review    │ ✅ Done
──────────────┼──────────────┼──────────────┼──────────────┼──────────────┼──────────────
US-001        │ US-003       │ US-002       │              │              │
US-004        │              │ US-005       │              │              │
US-006        │              │              │              │              │
```

---

## 📋 Backlog

### US-001: [Título]
- **Prioridad:** 🔴 Alta | 🟡 Media | 🟢 Baja
- **Estimación:** [S/M/L/XL]
- **Dependencias:** [ninguna | US-XXX]
- **Criterios de aceptación:**
  - [ ] [criterio 1]
  - [ ] [criterio 2]

---

[...más stories...]

---

## 📈 Métricas

| Métrica | Valor |
|---------|-------|
| Total stories | X |
| Completadas | Y |
| En progreso | Z |
| Velocity (stories/sprint) | — |
| Bloqueadas | 0 |
```

### Paso 4: Actualizar Tablero

Si el KANBAN.md ya existe, pregunta:

```
📋 Kanban existente detectado con [X] stories.

¿Qué quieres hacer?
1. 📊 Ver estado actual
2. ➡️  Mover una story (cambiar columna)
3. ➕ Agregar nueva story
4. 🔄 Sincronizar con User Stories (re-importar)
5. 📈 Ver métricas de progreso
```

### Reglas de Movimiento

- Una story solo avanza si cumple los criterios de la columna actual:
  - **Backlog → Planning:** Alguien la tomó para diseñar
  - **Planning → Dev:** Tiene spec/diseño aprobado
  - **Dev → QA:** Código completo, sin TODOs
  - **QA → Review:** Tests pasan, build exitoso
  - **Review → Done:** Aprobada y mergeada/deployada

- Las stories pueden retroceder:
  - **QA → Dev:** Bug encontrado
  - **Review → Dev:** Cambios solicitados

### Paso 5: Integración con La Forja

Si se detecta que el proyecto usa sandboxes (La Forja), actualizar automáticamente:
- Cuando un agente completa una fase → mover stories relacionadas
- Agregar columna de "Sandbox #N" si hay múltiples agentes

### Paso 6: Generar Dashboard HTML

Después de crear o actualizar `KANBAN.md`, SIEMPRE generar el dashboard visual HTML.

Crear el archivo `.claude/reports/kanban-board.html` — un HTML standalone (sin dependencias externas) con:

**Layout:** Board estilo Trello con 6 columnas lado a lado
- Cada columna tiene header con emoji + nombre + count de stories
- Cada story es una card con: ID, título, prioridad (color-coded), estimación
- Cards tienen borde izquierdo coloreado por prioridad: rojo (Alta), amarillo (Media), verde (Baja)

**Estilos (inline CSS, sin frameworks):**
- Fondo: `#0079bf` (azul Trello) para el header, `#f4f5f7` para el body
- Columnas: `#ebecf0` con border-radius, min-width 250px, flex layout horizontal con scroll
- Cards: blancas con sombra sutil, padding 12px, margin 8px, border-radius 8px
- Responsive: en mobile las columnas van en stack vertical
- Barra de progreso visual en el header (% completado)

**Datos:** Leer directamente del contenido de `KANBAN.md` que acabas de generar/actualizar. Parsear:
- Cada sección (## Backlog, ## Desarrollo, etc.) para extraer stories por columna
- Las métricas para la barra de progreso
- Prioridad y estimación de cada story para las cards

**Footer:** Timestamp de última actualización + "Generated by Forge /kanban"

**El usuario abre este HTML en su browser para ver el estado visual.** No necesita interactuar — es read-only. El agente lo regenera cada vez que actualiza `KANBAN.md`.

## Output

- Crea o actualiza `KANBAN.md` en la raíz del proyecto
- Genera `.claude/reports/kanban-board.html` (dashboard visual)
- Muestra resumen visual al usuario después de cada cambio
- Informa: `Dashboard visual generado en .claude/reports/kanban-board.html — ábrelo en tu browser.`

## Integración con El Yunque (Build Manual)

El Yunque actualiza automáticamente el Kanban al transicionar entre fases (PASO 4).
El agente lee `KANBAN.md`, mueve las stories correspondientes, y regenera el HTML.
El usuario puede abrir el HTML en cualquier momento para ver el estado actual.
